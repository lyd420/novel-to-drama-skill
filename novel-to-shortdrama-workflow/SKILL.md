# 中文小说到动漫短剧工作流

一键式完成中文小说的海外本地化、英文翻译和短剧分集改编，生成适合海外市场的动漫短剧内容。

## 工作流概览

```
┌─────────────────────────────────────────────────────────────┐
│  输入：中文小说文件 (.txt/.md)                                │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  STEP 1: 海外本地化 (novel-overseas-localization)           │
│  - 价值观红线检查                                           │
│  - 中式文化符号本地化                                       │
│  - 剧情逻辑适配                                             │
│  - 中式表达清洗                                             │
│  输出：修改后中文版 + 分析报告                              │
└──────────────────────────┬──────────────────────────────────┘
                           │ 用户确认后继续
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  STEP 2: 英文翻译 (novel-translation-helper)                │
│  - 选择地名本地化策略（美国/英国/通用）                     │
│  - 提取术语表（人名→英文姓名）                              │
│  - 提取术语表（地名→美国地名）                              │
│  - 全文翻译                                                 │
│  输出：术语表 + 英文版小说                                  │
└──────────────────────────┬──────────────────────────────────┘
                           │ 用户确认后继续
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  STEP 3: 短剧分集 (short-drama-adapter)                     │
│  - 分析原文结构                                             │
│  - 设计短剧节奏                                             │
│  - 拆分为1-3分钟分集                                        │
│  - 生成详细分集脚本                                         │
│  输出：分集总表 + EP1-N详细脚本                             │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  最终输出                                                   │
│  - 修改后中文版                                             │
│  - 英文版小说                                               │
│  - 分集脚本                                                 │
│  - 完整工作流报告                                           │
└─────────────────────────────────────────────────────────────┘
```

### 第二步：执行STEP 1 - 海外本地化（分析阶段，非翻译）

**重要说明**：本步骤只做**中文分析和建议**，**不进行英文翻译**。

自动调用 `novel-overseas-localization` skill：

1. **读取原文**
2. **四步检查**（输出都是中文）：
   - 海外价值观红线检查
   - 中式文化符号本地化
   - 剧情逻辑商业化适配
   - 中式表达清洗
3. **生成修改后的中文版**
4. **输出分析报告**（中文）

**STEP 1 vs STEP 2 分工**：
| STEP 1（本步） | STEP 2（下一步） |
|----------------|------------------|
| ✅ 中文分析报告 | ✅ 英文翻译 |
| ✅ 标记需翻译的人名 | ✅ 确定英文姓名 |
| ✅ 标记需本地化的地名 | ✅ 确定英文地名 |
| ❌ 不做英文翻译 | ❌ 不做中文分析 |

**输出文件**（均为中文）：
- `[文件名]_本地化.md` - 修改后的**中文版**
- `[文件名]_分析报告.md` - 详细分析报告（中文）

**用户确认点（所有模式必须执行）**：
```

  └─ [文件名]_分析报告.md  ← 四步分析详情

【关键说明】
• 本步骤只做中文分析，未进行英文翻译
```

- **一键执行模式**：显示简化确认（统计信息 + "确认继续？"），但仍需用户确认

### 第三步：执行STEP 2 - 英文翻译（关键质量控制点）

自动调用 `novel-translation-helper` skill：

1. **地名策略确认**（如用户未指定，则询问）
2. **术语表提取**：
   - 人物名称 → **英文姓名（First+Last Name，禁止拼音）**
   - 地名 → **本地化地名（禁止保留中文拼音）**
   - 家族姓氏 → **英文姓氏**
   - 公司/组织 → **英文命名**
3. **用户确认术语表**
4. **全文翻译**（严格执行以下红线）

**🚨 STEP 2 核心质量红线**：

| 红线要求 | 错误示例 | 正确示例 |
|----------|----------|----------|
| **禁止拼音姓名** | ❌ Xiao Nian Gu / Gu Xiao Nian | ✅ Stella Gray |
| **禁止中文残留** | ❌ "Chapter 第一章" | ✅ "Chapter 1" |
| **禁止中文称呼** | ❌ Mr. Gu / Miss Fu | ✅ Mr. Gray / Ms. Foster |
| **禁止拼音地名** | ❌ Beijing, Shanghai | ✅ Washington D.C., New York |
| **禁止混合格式** | ❌ Stella (晓念) | ✅ Stella |

**翻译后强制自检**：
- [ ] 全文搜索中文字符（正则 `[一-鿿]`），结果必须为 0
- [ ] 检查所有人名格式为 First Last，无拼音残留
- [ ] 检查所有称呼为 Mr./Ms./Mrs. + 英文姓氏
- [ ] 检查所有地名为英文地名，无拼音

**输出文件**：
- `[文件名]_术语表_[日期].csv` - 术语对照表
- `[文件名]_EN.txt` - **纯英文**版小说（零中文、零拼音）

**用户确认点（分步模式必须执行）**：

## 参数配置

### 可配置参数

| 参数 | 选项 | 默认 | 说明 |
|------|------|------|------|
| `地名策略` | 美国/英国/通用/保留 | 美国 | 地名本地化方式 |
| `分集时长` | 1-2分钟/2-3分钟/混合 | 混合 | 短剧集长 |
| `冲突密度` | 高/中/低 | 高 | 每集冲突数量 |
| `完全本地化` | 是/否 | 是 | 是否消除中国痕迹 |

### 配置文件示例

```yaml
workflow_config:
  step1_localization:
    enabled: true
    redline_check: true
    cultural_adaptation: true
    
  step2_translation:
    enabled: true
    location_strategy: "usa_full"  # usa_full/uk/full/generic/keep
    name_translation: "full_english"  # first+last name
    
  step3_drama_adaptation:
    enabled: true
    episode_duration: "mixed"  # 1-2min/2-3min/mixed
    pacing: "high_conflict"  # high/medium/low
    
output:
  generate_report: true
  save_intermediate: true
```

## 故障排除

### 常见问题

**Q: STEP 1修改后需要大量人工调整？**  
A: 可以暂停工作流，修改 `[文件名]_本地化.md` 后，使用 `恢复工作流` 继续。

**Q: 术语表翻译不满意？**  
A: 在STEP 2的用户确认点，回复修改意见，如："修改：顾晓念 → Sophia Gray"

**Q: 分集节奏不符合预期？**  
A: 可以重新执行STEP 3，调整 `分集时长` 和 `冲突密度` 参数。

**Q: 处理大文件时内存不足？**  
A: 可以分段处理，或增加 `batch_size` 配置。

## 执行建议

- **大文件处理**：超过10万字的小说建议分卷处理
- **人工审核**：每个STEP的用户确认点建议仔细审核
- **迭代优化**：可多次运行工作流，逐步优化输出质量
- **版本管理**：建议保存每次运行的输出，便于对比
