# 📚 Novel to Short Drama Skills

一套将中文网络小说转换为海外短剧剧本的 **Kimi Code CLI 技能套件**。

> 从「霸道总裁文」到「TikTok 短剧」，一站式 AI 工作流

---

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🌍 **文化本地化** | 中国背景 → 美国/英国背景，符合海外观众认知 |
| 🏷️ **术语去重** | 自动检测历史使用过的姓氏/地名，避免重复 |
| 🎭 **短剧结构** | 自动生成 28 集短剧，每集 1-2 分钟，含卡点设计 |
| 🔄 **工作流编排** | 一键执行或分步审核，灵活可控 |

---

## 📦 技能组成

本套件包含 **4 个协同工作的技能**：

```
┌─────────────────────────────────────────────────────────────┐
│                    主控工作流 (Master)                       │
│            novel-to-shortdrama-workflow                      │
└───────────────────────┬─────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   STEP 1     │ │   STEP 2     │ │   STEP 3     │
│   合规审核    │ │   英文翻译    │ │   短剧分集    │
│ Localization │ │ Translation  │ │   Drama      │
└──────────────┘ └──────────────┘ └──────────────┘
```

### 1️⃣ novel-overseas-localization
**海外本地化与合规审核**

- 识别价值观红线内容（暴力、歧视、违禁情节）
- 中国地标 → 海外地标（如：北京 → 华盛顿）
- 中国姓氏 → 英文姓氏（如：顾 → Gray）
- 输出：合规中文版 + 分析报告 + 术语表

### 2️⃣ novel-translation-helper
**英文翻译与术语去重** ⭐核心

- 完整英文翻译（非拼音）
- **历史术语表检查**：自动避免与之前小说重复使用姓氏/地名
- 支持美国/英国/通用三种本地化风格
- 输出：英文版 + 更新后的术语表

### 3️⃣ short-drama-adapter
**短剧分集改编**

- 将小说改编为 28 集短剧
- 每集设计：钩子 → 冲突 → 反转 → 卡点
- 短剧专用改编：节奏快、冲突密、付费点强
- 输出：分集剧本 + 拍摄建议

### 4️⃣ novel-to-shortdrama-workflow
**主控工作流**

- 一键串联 3 个步骤
- 支持分步审核（每步可人工修改后再继续）
- 自动管理文件输入输出

## 🚀 快速开始

### 安装技能

将本仓库复制到 Kimi Code CLI 的 skills 目录：

```bash
# 克隆仓库
git clone https://github.com/lyd420/novel-to-drama-skill.git

# 复制到 skills 目录（根据你的实际路径调整）
cp -r novel-to-drama-skill/* ~/.config/agents/skills/
```

### 使用方法

#### 方式一：一键工作流（推荐）

在 Kimi Code CLI 中输入：

```
开始工作流，文件：我的小说.txt
```

或简写：

```
一键生成短剧：我的小说.txt
```

AI 会自动执行：
1. 读取小说 → 2. 合规审核 → 3. 翻译 → 4. 分集 → 5. 输出完整剧本

#### 方式二：分步执行（需要审核）

每完成一步，你可以检查修改后再继续：

```
# 第1步：本地化
使用 novel-overseas-localization 处理：我的小说.txt

# （检查输出后）
# 第2步：翻译
使用 novel-translation-helper 翻译：我的小说_本地化.md

# （检查输出后）
# 第3步：分集
使用 short-drama-adapter 改编：我的小说_EN.txt
```

## 📋 工作流程详解

### 输入
- 中文网络小说（`.txt` 格式）

### 输出（每本小说生成 4-5 个文件）

| 文件 | 说明 |
|------|------|
| `小说_本地化.md` | 合规修改后的中文版 |
| `小说_分析报告.md` | 红线检测 + 文化替换详情 |
| `小说_术语表.csv` | 中英术语对照（用于翻译） |
| `小说_EN.txt` | 完整英文版（美国/英国背景） |
| `小说_ShortDrama.md` | 28集短剧分集剧本 |

### 术语去重机制

**问题**：多本小说翻译时，如何避免「顾总」都叫 "Mr. Gray"？

**解决**：系统自动维护 `used-names.md` 历史术语表

```
新小说: 主角姓「顾」
   ↓
检查 used-names.md
   ↓
发现: 顾 → Gray 已被小说A使用 ⚠️
   ↓
自动建议替代:
   - 顾 → Grey (变体)
   - 顾 → Grayson (变体)
   - 顾 → Graham (不同姓)
   ↓
用户确认 → 翻译 → 更新术语表
```

**已占用姓氏示例**（新小说需避开）：
- Gray（顾）、Foster（傅）、Reyes（阮）、Walsh（王）

**可用姓氏示例**：
- Shaw（沈）、Lewis（陆）、Sutton（苏）、Hughes（许）

## 🎯 使用示例

### 示例 1：霸总追妻文

**输入**：`霸道总裁的契约娇妻.txt`

**AI 处理**：
1. 审核：检查是否有过度暴力/强制爱情节
2. 本地化：
   - 顾氏集团 → Gray Corporation (华盛顿)
   - 厉景深 → Ethan Sterling
   - 小白花女主 → Lily Hart
3. 翻译：完整英文版
4. 分集：
   - 第1集：替嫁入局（钩子：发现新郎是死对头）
   - 第7集：宴会羞辱（卡点：揭开身份之谜）
   - 第28集：真相大白（大结局）

**输出**：可直接用于拍摄的 28 集短剧剧本

## ⚙️ 高级配置

### 修改默认设置

编辑 `novel-to-shortdrama-workflow/references/workflow-config.md`：

```markdown
# 可配置项
- 默认本地化风格: 美国/英国/通用
- 短剧集数: 28集（标准）/ 50集（长篇）
- 每集时长: 1-2分钟 / 2-3分钟
- 术语冲突处理: 自动建议 / 强制停止询问
```

### 自定义术语表

编辑各技能下的 `references/` 文件：

- `english-names.md` - 添加更多姓氏映射
- `english-locations.md` - 添加更多城市
- `redlines.md` - 调整合规红线

## 📝 注意事项

1. **文件格式**：输入小说请使用 `.txt` 格式，UTF-8 编码
2. **章节标识**：最好有明确的章节标题（如「第一章」），便于分集
3. **术语审核**：关键术语（主角姓名）建议人工确认后再继续
4. **版权**：请确保您拥有小说的翻译和改编权利
